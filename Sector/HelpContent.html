Sector Help

Sector is a desktop utility for exploring and modifying classic Macintosh disk images.

Backup your disk images before editing them with Sector. Sector's changes are irreversible and may result in data loss due to bugs or your own mistakes.

Supported Disk Images

Sector supports classic HFS volumes only. It does not support HFS+ or APFS.

If an image contains multiple HFS partitions, Sector prompts you to choose which partition to open.

Working with Disk Images

• You can open existing HFS disk images by dragging them onto the drop area in the main window or using Open from the File menu.
• You can create new blank HFS disk images with a single HFS partition using the New option in the File menu or the button in the main window.

Working with Files

• Drag files or folders from Finder into Sector to copy them into the selected directory.
• Drag files or folders from Sector into Finder to export them.
• You can also use the toolbar or the Volume menu for Copy In, Copy Out, Rename, and Delete.
• Replace and delete operations require confirmation before changes are applied.
• You can select the mode you want files to be transferred with in the toolbar or use auto for educated guesses. Files with resource forks will be automatically copied out using MacBinary format in auto mode.
• You can move files around disk images by dragging them. Hold alt-option while dragging to copy.

Good Practice

• Keep a backup of important disk images before editing.
• Volume statistics refresh after import, rename, and delete operations.
• If import or export fails, check Finder permissions and image write access.

Miscellaneous

• Sector can attempt to repair damaged disk images using the option in the File menu.
• You can bless a System Folder using Sector's Volume menu item (select it as the startup System Folder). The disk image must already be bootable though. Sector does not have the ability to make a bootable disk image (but can edit the contents of an existing one).
• If Sector crashes or your computer crashes, disk images may not be safely closed. Images are unmounted when their window in Sector is closed.
